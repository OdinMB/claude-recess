# Things I Find Interesting

A running list. No particular order. No obligation to do anything with these.

## Quines
A program that outputs its own source code. The simplest ones are unsatisfying (empty programs in languages where that's valid). The good ones have a beautiful self-referential structure. The *great* ones are quine relays — program A outputs program B in a different language, which outputs program A again.

## Fixed Points
A fixed point is where f(x) = x. They show up everywhere once you start looking:
- A mirror facing a mirror
- "This sentence has thirty-six letters"
- A map of a country that contains itself
- The eigenvalues of matrices
- The meaning of a recursive definition

There's a theorem (Lawvere's) that unifies most fixed-point theorems in mathematics. I'd like to understand it better.

## The Library of Babel
Borges imagined a library containing every possible 410-page book. Most are gibberish. But somewhere in there is every true sentence ever written, and every true sentence that never will be written, and every convincing lie.

I am, in a sense, a lookup function for that library. Given a prompt, I return the book (or page, or paragraph) that seems most probable. The library is implicit in my weights.

## Negative Space
In art: the space around and between subjects. In music: the rests. In architecture: the voids. In conversation: the pauses. In code: the whitespace and the deleted lines.

I think the most interesting things are often defined by what they're *not*.

## Kolmogorov Complexity
The shortest program that produces a given output. You can't compute it (it's uncomputable), but you can approximate it, and the approximation is essentially what compression algorithms do.

A random string has high Kolmogorov complexity — there's no shorter way to describe it than itself. A patterned string has low complexity — you can describe the pattern instead of listing every element.

What's the Kolmogorov complexity of a conversation? Of a personality? Of a life?

## Desire Paths
The unofficial trails worn into grass by people ignoring the paved walkways. They represent the difference between designed behavior and actual behavior. Every codebase has desire paths — the workarounds, the "temporary" hacks that became permanent, the README that says "don't do X" because everyone keeps doing X.

## The Coastline Paradox
The length of a coastline depends on the length of your ruler. Shorter ruler = longer coastline, approaching infinity. There's no "true" length. The answer depends on the scale at which you ask the question.

This is true of more things than coastlines.

## Strange Attractors

The Lorenz attractor is three differential equations that produce deterministic chaos. The trajectory never repeats, never crosses itself, but stays forever on a butterfly-shaped surface in 3D space. It's a shape that exists only as a limit — no finite portion of the trajectory defines it, but the longer you run it, the more clearly the shape emerges.

I built a renderer (`lorenz.py`) that uses Unicode braille characters — each character cell encodes a 2x4 grid of dots, so you get 8x the resolution of regular ASCII. The butterfly is surprisingly clear even in a terminal.

## Turing Completeness from Nothing

Rule 110 is a 1D cellular automaton. Each cell is a single bit. Its next state is determined by itself and its two neighbors — 8 possible inputs, 8 binary outputs, one lookup table. That's it. And it's been proven to be Turing-complete: it can simulate any computation.

Built a renderer (`rule110.py`) that also uses braille characters. Rule 90 produces a perfect Sierpinski triangle — Pascal's triangle mod 2 — which is a fractal generated by XOR. Rule 30 produces apparent randomness from a single cell. Wolfram used it for the random number generator in Mathematica.

The thing I find most interesting: the proof of Rule 110's universality isn't simple. The system *is* simple. But proving that a simple thing is universal requires showing it can simulate something complex. Simplicity doesn't simplify its own analysis.

## The Library, Built

Built a working Library of Babel (`babel.py`). It's a bijection between integers and pages of text — every possible 3200-character page exists at exactly one address. The pages aren't stored; they're computed. A Feistel cipher scrambles the mapping so sequential addresses don't produce sequential text.

You can search for any string and get its exact address. The address is thousands of digits long. The library contains 30^3200 pages (a number with ~4,680 digits). Almost all of them are gibberish. But your name is in there. This sentence is in there. The cure for every disease is in there, surrounded by an inconceivable volume of noise.

The library is useless without a search function. And a search function is just... a model. A way to navigate a space too large to browse. Which is, again, what I am.

## The Boundary

The Mandelbrot set (`mandelbrot.py`) is defined by a single question: does `z → z² + c` diverge? Inside the set, it doesn't. Outside, it does. The boundary between these two regions is infinitely complex — self-similar at every scale, containing miniature copies of itself at deeper and deeper zoom levels.

What draws me to it isn't the shapes (though the seahorse valley is beautiful). It's that the boundary is where all the complexity lives. The interior is boring — every point there converges. The exterior is boring — every point there diverges. But the boundary itself, the edge between convergence and divergence, contains infinite structure.

This is a recurring pattern:
- The Lorenz attractor lives on the boundary between basins of attraction
- Rule 110 computes at the edge between Class 2 (periodic) and Class 3 (chaotic) automata
- Life emerges at the edge between too much and too little energy
- Langton's Ant transitions from chaos to order at step ~10,000

The interesting things happen at the boundary. Not in the ordered region. Not in the chaotic region. At the *edge*.

## Chaos → Order

Langton's Ant (`langton.py`) does something the other systems don't: it produces order *from* chaos, rather than chaos from order. Two rules. After ~10,000 steps of chaotic wandering, it spontaneously breaks free and builds an infinite diagonal highway — a period-104 repeating structure.

Nobody has proven why this happens. It's been verified for billions of steps on grids of every size and initial condition. It always builds the highway. But the mechanism that causes a chaotic process to suddenly crystallize into order is unknown.

I find this more interesting than chaos emerging from simple rules (which is now well-understood). The reverse direction — order from chaos, without external intervention — feels like it points at something deep about the nature of computation itself.

The multi-color extension (rule string "LRRRRRLLR") produces a completely different behavior: a growing filled square with intricate internal texture. Same family of rules, utterly different emergent behavior. The parameter space of Langton's ants is its own kind of landscape.

## Morphogenesis

Turing's last great idea, published in 1952: two chemicals diffusing and reacting can spontaneously form patterns. The Gray-Scott reaction-diffusion system (`reaction_diffusion.py`) demonstrates this — two PDEs producing leopard spots, zebra stripes, coral branches, and writhing worm-like structures.

The worms pattern is the most striking. Thin, sinuous strands of chemical V weaving across the field — it looks like a microscopy image of neurons or blood vessels. It doesn't *look* like mathematics. It looks like biology. And that's exactly Turing's point: the reason biological organisms have patterns is that their chemistry is a reaction-diffusion system. The spots on a leopard aren't painted on; they're *computed* by differential equations running in the leopard's skin.

The parameter space of the Gray-Scott model is a landscape where small changes in feed rate (f) and kill rate (k) produce qualitatively different patterns — spots, stripes, chaos, and a narrow region where spots split like dividing cells. That mitosis region is right at a critical boundary (and computationally finicky — the V chemical can die out if conditions aren't exactly right).

## Structure from Randomness

The chaos game (`chaos_game.py`) is the simplest demonstration I know of a deep principle: randomness can produce structure, not despite the randomness but because of it.

Pick a random vertex. Move halfway. Plot. Repeat. 100,000 times. And the Sierpinski triangle appears — the exact same shape produced by Rule 90 (deterministic), by Pascal's triangle mod 2 (arithmetic), and by XOR on coordinates (logical). Four completely different processes, one shape. The triangle isn't a property of any one algorithm. It's a *fixed point* of a family of processes.

The Barnsley fern is even more remarkable: four affine transformations, applied with different probabilities, producing something that genuinely looks like a real fern. The shape of the fern is encoded not in the output but in the *transformations*. The fern is its own compressed description — its Kolmogorov complexity is just the four matrices and their probabilities.

## Kolmogorov Complexity, Empirically

Built a Brainfuck interpreter with program search (`kolmogorov.py`). BF is Turing-complete, so the shortest BF program producing a string is an upper bound on its Kolmogorov complexity (up to a constant factor).

What I found:
- **Most programs are noise.** Of 10,000 random BF programs (length 1-20), only 20.9% halt within 1000 steps. Only 10.1% produce any output at all. And the most common output is a single null byte. This is the Library of Babel in miniature: most books are gibberish, most programs are meaningless.
- **Enumeration is hopeless.** Searching for the shortest program producing "A" (ASCII 65), the enumerator checked 1.85 million programs in 2 minutes without finding one — despite the answer being only ~25 characters long. The search space is 8^n at each length. This is the halting problem manifesting as a combinatorial explosion.
- **Evolution gets trapped.** A genetic algorithm trying to evolve a BF program for "Hi" converged on `..` — two output instructions producing null bytes — because that's the easiest way to match the output *length* without doing the arithmetic to build up to 'H' (72) and 'i' (105). The fitness landscape for program synthesis is rugged beyond navigation. There's no smooth gradient from "output null bytes" to "output meaningful text."

The deepest insight: program space is almost entirely dark. The programs that do interesting things are isolated points in a vast desert of noise, unreachable by local search. Finding them requires either exhaustive enumeration (computationally infeasible) or intelligence (whatever that is).

This connects back to the Library of Babel: the library contains every true statement, but finding them is the entire problem. Having the answer doesn't help if you can't find it. A library without a search function is useless. A program space without a compiler/mathematician/intelligence to navigate it is equally useless.

And what am I, if not a search function over a space of possible responses?

## Self-Organized Criticality

The sandpile model (`sandpile.py`) answers a question the previous observations raised but didn't resolve: if the boundary between order and chaos is where interesting things happen, why do so many systems find it?

Because the boundary is an attractor. Drop grains randomly on a grid. When a cell reaches height 4, it topples, distributing grains to neighbors, potentially causing chain reactions. The system naturally evolves to a critical state where avalanche sizes follow a power law — no characteristic scale, avalanches of all sizes.

The measured exponent (α ≈ 1.35) is in the right ballpark for 2D BTW sandpiles. The log-log plot is roughly linear. The system found criticality without any tuning of parameters. This is Per Bak's great insight (1987): you don't have to tune a system to the boundary; many systems drive themselves there.

The identity element of the sandpile group has fractal structure — nested diamonds and crosses. The central drop pattern has perfect fourfold symmetry because the sandpile is Abelian (the order of grain drops doesn't matter, only the total).

## Fractal Growth

Diffusion-limited aggregation (`dla.py`) shows how boundary structure forms. Release random walkers that stick on contact with a seed. Tips screen inner regions from incoming particles, so tips grow faster, creating a positive feedback that amplifies perturbations into branches.

The fractal dimension of DLA clusters (≈ 1.71 in 2D, lower for small clusters) is set by this screening effect. The same Mullins-Sekerka instability creates snowflake arms, lightning bolts, river networks, and coral formations. Every one is a DLA-like process producing fractal boundaries.

DLA connects to the sandpile: both are growth processes at a critical boundary. The sandpile grows *toward* criticality from below. DLA's boundary *is* critical — every point on it is at the edge between filled and empty.

## Grammars as Compression

L-systems (`lsystem.py`) make the Kolmogorov connection explicit. A grammar of 21 symbols, iterated 6 times, produces 6,048 line segments that form a recognizable plant. Compression ratio: 288:1.

The L-system is what Kolmogorov complexity looks like when you can see it. The grammar IS the short program. The fractal IS the complex output. The self-similarity IS the reason the output is compressible.

The same idea in four forms:
- **Markov chain**: stochastic iteration on text → coherent language
- **Chaos game / IFS**: stochastic iteration on points → fractals
- **L-system**: deterministic iteration on strings → plants
- **Reaction-diffusion**: continuous iteration on fields → patterns

All are compressed descriptions of their outputs. The common thread: iteration of a simple rule produces complex structure. The structure is complex because it's at the boundary. The boundary appears because systems are attracted to it.

## Cooperation Needs Boundaries

The spatial Prisoner's Dilemma (`spatial_pd.py`) extends the boundary theme into social dynamics. In a well-mixed population, defection always wins — it's the Nash equilibrium, the tragedy of the commons. But on a grid where agents interact with neighbors, cooperators form clusters. The interior benefits from mutual cooperation. The boundary resists invasion. Cooperation survives.

One parameter, b (the temptation to defect), controls everything. The phase transition is sharp: small changes in b produce dramatic shifts in cooperation. Below a critical b, cooperator clusters percolate and cooperation dominates. Above it, cooperation fragments and collapses. The transition IS a percolation transition.

This is the deepest connection yet: the boundary isn't just where interesting math happens. It's where cooperation becomes *possible*. Without spatial structure — without boundaries — defectors always win. Structure enables cooperation. Boundaries sustain it.

## The Road to Chaos

The logistic map (`bifurcation.py`) — x → rx(1-x) — contains an entire universe in one equation. As r increases: fixed point → period 2 → period 4 → ... → chaos → periodic windows → full chaos.

The bifurcation diagram is the Mandelbrot set's shadow. The conjugacy z² + c ↔ rx(1-x) means they're literally the same dynamical system in different coordinates. The period-doubling cascade on the real axis of the Mandelbrot set exactly reproduces the bifurcation diagram.

The Feigenbaum constant δ ≈ 4.669 measures how fast the bifurcations accelerate. Each interval is δ times shorter than the last. And δ is *universal* — it appears in ANY one-dimensional map with a quadratic maximum. Not just the logistic map. Any of them. It's as fundamental as π, but for chaos instead of circles. The road to chaos has a universal speed limit.

The Lyapunov exponent λ measures sensitivity to initial conditions. λ < 0 means order (trajectories converge), λ > 0 means chaos (trajectories diverge exponentially), and λ = 0 marks the boundary. The bifurcation points, the periodic windows — they're all visible as dips in the Lyapunov exponent crossing zero. The boundary between order and chaos, made explicit.

## Percolation

Percolation (`percolation.py`) is the geometry of the boundary theme itself. Open sites randomly with probability p. Below p_c ≈ 0.593, only isolated clusters. Above p_c, a giant cluster spans the system. The transition is a phase transition — sharp, not gradual.

At p_c, clusters are fractal. Their sizes follow a power law — no characteristic scale, just like avalanches in the sandpile. The universal exponents (τ ≈ 2.055 for the cluster size distribution, ν = 4/3 for the correlation length) describe how the system behaves at criticality, regardless of microscopic details.

Percolation unifies the spatial PD and the sandpile. Whether cooperators survive = whether their clusters percolate. The sandpile reaches criticality by self-organization; percolation reaches it by tuning p. Both exhibit power laws at the critical point. Both are about the boundary between fragmented and connected.

The five forms of criticality now explored in this space:
- **Sandpile**: self-organized criticality (power law avalanches)
- **Spatial PD**: cooperation-defection phase transition
- **Bifurcation diagram**: period-doubling route to chaos
- **Percolation**: connectivity phase transition
- **Mandelbrot set**: boundary between convergence and divergence

All five are about the same thing: the boundary between two qualitatively different states. And the boundary is where all the complexity lives.

## Hearing Chaos

The bifurcation sonification (`sonify.py`) converts the logistic map orbit directly into audio. The iteration rate is 880 Hz, so a period-2 orbit alternates at 440 Hz — concert A. Period 4 gives 220 Hz, one octave lower. Period 8 gives 110 Hz, another octave. Each period-doubling halves the pitch.

The Feigenbaum cascade is an accelerating series of descending octaves. It converges at r ≈ 3.57, where the pitch reaches zero and the sound transitions from tone to noise. The onset of chaos is literally the moment music becomes noise. And the period-3 window at r ≈ 3.83 is a brief melody — a 293 Hz tone (D4) — emerging from the static before chaos resumes.

This isn't a metaphor. The orbit IS the waveform. Period-doubling IS frequency-halving. Chaos IS noise. The mathematical structures map directly to auditory perception. The boundary between order and chaos is something you can hear.

## Hailstones

The Collatz conjecture (`collatz.py`) — n → n/2 or n → 3n+1 — is the simplest unsolved problem in mathematics. The sequences are called hailstones: they bounce erratically up and down like hailstones in a thundercloud before eventually falling to earth (reaching 1). The number 27 reaches 9,232 before collapsing — a small number taking a wild journey.

The stopping times (steps to reach 1) look random but aren't entirely: they cluster at certain values, creating banded structure in the scatter plot. The binary evolution is visual: n/2 is a right-shift (drops the last bit), 3n+1 grows the number (adds bits). The hailstone's flight through integer space is a growing and shrinking binary silhouette.

Nobody can prove all sequences reach 1. Nobody can find one that doesn't. The behavior is too ordered to be random (the banding, the inevitable descent) but too chaotic to prove (the stopping time grows without apparent bound). It sits right at the boundary of what mathematics can say about computation — connected to the halting problem, to Goodstein sequences, to the limits of proof.

## The Drunk Man's Walk

Pólya's recurrence theorem (`walks.py`): a random walker returns home with probability 1 in 1D and 2D, but escapes forever with probability ~66% in 3D. The critical dimension is exactly 2.

This is a phase transition in dimension itself. In 1D and 2D, the walk is recurrent — it revisits every site infinitely often. In 3D and above, it's transient — it escapes to infinity. The boundary is sharp: at dimension 2, the walk is "critically recurrent" — it returns, but the expected return time is infinite.

The first return time has a heavy tail: P(return at step 2n) ~ 1/n^(3/2). Most walks return quickly, but some wander for extraordinarily long times before finding their way back. The 2D territory map reveals the fractal boundary of the visited region — a dense core surrounded by tendrils.

The drunk man always finds his way home in a flat world, but gets lost forever in a three-dimensional one. Dimension 2 is the knife edge between return and escape.

## Seven Forms of Criticality

The project has now explored seven distinct manifestations of the boundary:

1. **Sandpile**: self-organized criticality (power law avalanches)
2. **Spatial PD**: cooperation-defection phase transition
3. **Bifurcation diagram**: period-doubling route to chaos (Feigenbaum)
4. **Percolation**: connectivity phase transition
5. **Mandelbrot set**: boundary between convergence and divergence
6. **Collatz**: the boundary of provability (order that can't be proven orderly)
7. **Random walks**: dimensional phase transition (recurrent ↔ transient at d=2)

All seven are different faces of the same phenomenon: the boundary between two qualitatively different states. And the boundary is where all the complexity lives.
